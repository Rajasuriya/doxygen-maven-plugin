397      * The SYMBOL_CACHE_SIZE determines the size of the internal cache use to
398      * determine which symbols to keep in memory and which to flush to disk.
399      * When the cache is full, less often used symbols will be written to disk.
400      * For small to medium size projects (&lt;1000 input files) the default
401      * value is probably good enough. For larger projects a too small cache size
402      * can cause doxygen to be busy swapping symbols to and from disk most of
403      * the time causing a significant performance penality. If the system has
404      * enough physical memory increasing the cache will improve the performance
405      * by keeping more symbols in memory. Note that the value works on a
406      * logarithmic scale so increasing the size by one will rougly double the
407      * memory usage. The cache size is given by this formula:
408      * 2^(16+SYMBOL_CACHE_SIZE). The valid range is 0..9, the default is 0,
409      * corresponding to a cache size of 2^16 = 65536 symbols
410      *
411      * @parameter expression="${doxygen.symbolCacheSize}" default-value="0"
412      */
